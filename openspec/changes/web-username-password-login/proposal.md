## Why

产品当前仅支持微信登录；为便于开发调试与内测场景，需要为 Web 端增加用户名密码登录能力，使团队与内测用户可在无微信环境下使用系统。同时需提供初始账号（user/1）以便开箱即用。

## What Changes

- 在 Web 端增加登录页：支持用户名 + 密码输入并提交登录。
- 后端 auth 模块增加用户名密码校验：接受用户名/密码，校验通过后签发会话（与现有微信登录共用会话模型）。
- 提供初始用户：用户名 `user`，密码 `1`（通过 seed/迁移或配置注入，仅用于开发/内测）。
- 未登录访问需登录才能访问的页面时，重定向到登录页；登录成功后跳回原目标或默认首页。
- **非目标**：不替代微信登录，不对外公开注册；用户名密码登录仅作为补充方式。

## Capabilities

### New Capabilities

- `web-username-password-auth`: Web 端用户名密码登录流程（登录页 UI、登录 API、会话签发、路由守卫、初始用户 user/1）。

### Modified Capabilities

- 无（现有 auth 为微信登录，本次为新增一种登录方式，不修改既有能力的需求定义）。

## Impact

- **apps/web**：新增或扩展 `views/auth` 登录页、`stores/auth` 登录态与登录接口调用、`router` 守卫。
- **apps/server**：`modules/auth` 增加用户名密码校验与 token 签发、用户表或凭证存储（若尚无则需模型扩展）、初始用户 seed。
- **packages/shared**：如需，增加登录请求/响应 DTO。
- **依赖**：后端需支持密码安全存储（如 bcrypt），不新增前端重型依赖。

## Non-goals

- 不提供公开注册、忘记密码、多因素认证。
- 不改变现有微信登录流程与 API。
- 初始用户 user/1 仅用于开发/内测，生产环境可通过配置关闭或移除。
