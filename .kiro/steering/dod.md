# 完工标准（Definition of Done）

每个任务完成前必须满足以下所有条件：

## 实现质量

- 所有功能必须真实实现，禁止使用 Mock 替代业务逻辑
- 无遗留 `// TODO` 或 `// FIXME` 注释
- 无 TypeScript `any` 类型，所有接口有明确类型定义
- 无 ESLint 错误或警告

## 测试要求

- 单元测试行覆盖率 ≥ 80%，通过率 = 100%
- 接口测试和功能测试通过率 = 100%
- API 接口测试禁止使用 Mock（必须调用真实实现）
- 接口测试、功能测试时连接真实 PostgreSQL（禁止内存数据库或 Mock 数据层）
- 接口测试、功能测试禁止 Mock 业务逻辑层

## 适用范围说明

> 单元测试中允许 Mock 外部依赖（微信 API、AI 接口、第三方文献 API、微信支付 API），
> 但接口测试和功能测试层必须使用真实数据库和真实业务逻辑。
