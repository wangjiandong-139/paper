# 用户认证与数据安全 需求质量检查清单

**目的**：验证微信登录、多端数据互通、权限控制相关需求的完整性与安全性
**创建日期**：2026-03-15
**功能规格**：[requirements.md](../requirements.md)
**覆盖需求**：需求 11（用户身份认证）、需求 1（向导持久化）、需求 8（运营后台）

---

## 需求完整性

- [ ] CHK001 - 是否定义了 JWT token 的有效期与刷新策略需求（token 过期后用户体验如何保持）？[Gap]
- [ ] CHK002 - 是否定义了运营后台的身份认证需求（运营人员如何登录管理后台，是否与用户账号体系隔离）？[Gap, Spec §需求8]
- [ ] CHK003 - 是否定义了用户账号注销需求（用户希望删除账号及所有数据时的处理流程）？[Gap]
- [ ] CHK004 - 是否定义了微信 UnionID 与 OpenID 的使用场景区分需求（何时用 OpenID，何时需要 UnionID）？[Completeness, Spec §需求11-AC2]
- [ ] CHK005 - 是否定义了 PC 端扫码登录二维码的有效期与过期处理需求（二维码过期后如何提示用户刷新）？[Gap]
- [ ] CHK006 - 是否定义了用户数据的访问权限边界需求（用户 A 是否能访问用户 B 的草稿/订单）？[Gap]
- [ ] CHK007 - 是否定义了运营后台 API（`/api/admin/*`）的访问控制需求（角色权限、IP 白名单等）？[Gap, Spec §需求8-AC2]

## 需求清晰度

- [ ] CHK008 - 需求 11 AC1「PC 端提供微信扫码登录」是否明确了扫码登录的完整交互流程（二维码展示位置、扫码后页面行为）？[Clarity, Spec §需求11-AC1]
- [ ] CHK009 - 需求 11 AC5「不允许未登录用户进入写作流程」是否明确了「进入写作流程」的具体路由范围？[Clarity, Spec §需求11-AC5]
- [ ] CHK010 - 需求 1 AC7「草稿永久保留，直到用户主动删除」是否明确了「主动删除」的操作入口与确认机制？[Clarity, Spec §需求1-AC7]
- [ ] CHK011 - 需求 11 AC3「多端数据互通」是否明确了数据同步的实时性要求（实时同步 / 最终一致性 / 手动刷新）？[Clarity, Spec §需求11-AC3]
