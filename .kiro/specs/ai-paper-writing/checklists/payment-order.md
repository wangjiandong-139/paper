# 支付与订单流程 需求质量检查清单

**目的**：验证支付与订单相关需求的完整性、清晰度与一致性
**创建日期**：2026-03-15
**功能规格**：[requirements.md](../requirements.md)
**覆盖需求**：需求 5（步骤 4 预览与支付）、需求 6（步骤 5 后台生成与订单管理）

---

## 需求完整性

- [ ] CHK001 - 需求 5 AC3 中「完成支付或点击确认并生成」是否明确区分了「先支付后生成」与「直接确认」两种路径的触发条件？[Ambiguity, Spec §需求5-AC3]
- [ ] CHK002 - 是否定义了支付超时场景（用户打开支付弹窗后长时间未操作）的处理需求？[Gap]
- [ ] CHK003 - 是否定义了微信支付回调重复通知（幂等性）的处理需求？[Gap]
- [ ] CHK004 - 是否定义了订单从 PENDING_PAYMENT 到 GENERATING 状态转换的触发条件与时序？[Completeness, Spec §需求6-AC1]
- [ ] CHK005 - 是否定义了用户在支付页面关闭后重新进入时的状态恢复需求（订单已创建但未支付）？[Gap]
- [ ] CHK006 - 需求 5 AC6 中「不支持退款」是否定义了向用户展示该政策的时机与方式（如支付前告知）？[Completeness, Spec §需求5-AC6]
- [ ] CHK007 - 是否定义了订单列表的分页或加载策略（用户有大量历史订单时）？[Gap]
- [ ] CHK008 - 是否定义了 FAILED 状态订单的「重试」操作是否重新计费？[Gap]

## 需求清晰度

- [ ] CHK009 - 需求 5 AC2 中「展示价格与功能说明」是否明确了价格的展示格式（如单位：元/分）？[Clarity, Spec §需求5-AC2]
- [ ] CHK010 - 需求 6 AC3 中「已生成第 X 章」的进度格式是否足够明确（如「已生成 2/5 章」）？[Clarity, Spec §需求6-AC3]
- [ ] CHK011 - 需求 6 AC8 中「页面内状态更新提示」是否明确了提示的展示形式（Toast / Banner / 弹窗）？[Clarity, Spec §需求6-AC8]
- [ ] CHK012 - 需求 5 AC5 中「明确的失败提示」是否定义了需要展示的具体信息（错误码、用户可操作的下一步）？[Clarity, Spec §需求5-AC5]
- [ ] CHK013 - 需求 6 AC6 中「联系客服」入口是否与需求 2 AC6 中的客服入口实现方式保持一致（微信二维码弹窗）？[Consistency, Spec §需求6-AC6, §需求2-AC6]
