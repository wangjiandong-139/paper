# AI 生成与引用约束 需求质量检查清单

**目的**：验证 AI 论文生成、引用溯源约束相关需求的完整性、清晰度与可测量性
**创建日期**：2026-03-15
**功能规格**：[requirements.md](../requirements.md)
**覆盖需求**：需求 6（后台生成）、需求 10（引用约束与溯源校验）、需求 4（提纲）、需求 7（改稿）

---

## 需求完整性

- [ ] CHK001 - 是否定义了 AI 生成正文时「仅使用步骤 2 文献」约束的具体 prompt 结构要求，还是仅说明了约束目标？[Completeness, Spec §需求10-AC1]
- [ ] CHK002 - 是否定义了当 AI 生成内容中出现幻觉引用（文献列表中不存在的引用）时的处理需求（自动删除 / 标记 / 重新生成）？[Gap]
- [ ] CHK003 - 是否定义了提纲中「图/表/公式/代码占位」在生成时「留空」的具体格式需求（如占位符文本、样式）？[Completeness, Spec §需求4-AC5]
- [ ] CHK004 - 是否定义了 AI 生成章节失败后单章重试的需求（重试次数上限、重试触发方式）？[Gap]
- [ ] CHK005 - 是否定义了生成内容的语言一致性需求（步骤 1 选择英文时，生成内容是否全程英文）？[Gap]
- [ ] CHK006 - 是否定义了「降重」与「降 AI 痕迹」两项操作的效果验收标准（如降重后查重率目标值）？[Completeness, Spec §需求7-AC4]
- [ ] CHK007 - 是否定义了改稿操作「按意见修改」中修改意见的长度或格式约束？[Completeness, Spec §需求7-AC3]

## 需求清晰度

- [ ] CHK008 - 需求 10 AC2 中「确定无法溯源」与「可能匹配请确认」两类结果的判断标准是否有明确定义？[Clarity, Spec §需求10-AC2]
- [ ] CHK009 - 需求 6 AC2 中「无法与步骤 2 文献列表匹配的引用」的匹配算法是否有明确描述（精确匹配 / 模糊匹配 / 基于哪些字段）？[Clarity, Spec §需求6-AC2]
- [ ] CHK010 - 需求 4 AC4 中章节字数「范围 100～2000 字」是否明确了超出范围时的处理方式（阻止保存 / 警告提示）？[Clarity, Spec §需求4-AC4]
- [ ] CHK011 - 需求 7 AC2 中「AI 绘图接口生成图片并插入正文」是否明确了图片插入位置的需求（光标位置 / 章节末尾 / 占位符替换）？[Clarity, Spec §需求7-AC2]
- [ ] CHK012 - 需求 4 AC6 中「粘贴已有提纲」的解析规则是否有明确定义（支持哪些格式：数字编号 / 缩进 / Markdown）？[Clarity, Spec §需求4-AC6]

## 需求一致性

- [ ] CHK013 - 需求 6 AC1「按章依次生成」与需求 4 AC5「图/表/公式/代码占位」的关联是否一致（生成时如何处理占位标记）？[Consistency, Spec §需求6-AC1, §需求4-AC5]
- [ ] CHK014 - 需求 10 AC4 的 round-trip 属性（引用标注 → 文献列表查找 → 可溯源或标出）与需求 6 AC2 的引用校验描述是否完全对齐，无重复或矛盾？[Consistency, Spec §需求10-AC4, §需求6-AC2]
- [ ] CHK015 - 需求 7 AC3「按意见修改不修改图/表/公式/代码」与需求 7 AC2「加图/表」的边界是否清晰（已插入的图表是否受保护）？[Consistency, Spec §需求7-AC3, §需求7-AC2]
- [ ] CHK016 - 需求 7 AC1a 中「降重、降 AI 痕迹、扩写、缩写、润色」均计入改稿次数，与需求 7 AC4 中「降重与降 AI 痕迹」的描述是否完整覆盖所有计次操作？[Consistency, Spec §需求7-AC1a, §需求7-AC4]
