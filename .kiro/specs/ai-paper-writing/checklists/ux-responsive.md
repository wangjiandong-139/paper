# 移动端 UX 与响应式布局 需求质量检查清单

**功能规格**：[requirements.md](../requirements.md)

- [ ] CHK004 - 是否定义了步骤 6 改稿页移动端「底部抽屉」的触发方式需求（固定按钮 / 手势上滑）？[Completeness, Spec §需求7-AC1]

## 需求清晰度

- [ ] CHK008 - 需求 9 AC2「触摸热区不小于 44pt」是否明确了 pt 与 px 的换算基准（1pt = 1px / 2px / 设备像素比）？[Clarity, Spec §需求9-AC2]
- [ ] CHK009 - 需求 1 AC2「步骤名称可点击回看」是否明确了「回看」的交互行为（跳转到该步骤 / 展示摘要弹窗）？[Clarity, Spec §需求1-AC2]
- [ ] CHK010 - 需求 1 AC6「不允许跨步骤乱跳」的「乱跳」边界是否明确（如步骤 6 能否跳回步骤 3）？[Clarity, Spec §需求1-AC6]
- [ ] CHK011 - 需求 9 AC5「简化富文本工具栏」是否明确了保留哪些格式操作（加粗/斜体/标题/列表等）？[Clarity, Spec §需求9-AC5]
- [ ] CHK012 - 需求 2 AC8「大卡片或底部弹出列表」是否明确了两种展示方式的选择条件，还是由实现自行决定？[Clarity, Spec §需求2-AC8]
- [ ] CHK013 - 需求 1 AC4「仅显示一个主操作按钮」是否明确了辅助操作按钮（如「返回」「跳过」）的展示规则？[Clarity, Spec §需求1-AC4]
