# 傻瓜式「AI 写论文」应用设计建议

目标：**界面清晰、步骤最少、操作极简**，让用户像填表一样完成论文写作引导。**以移动端为主、同时支持 PC 端**，主流程在手机竖屏下即可完成。

---

## 一、整体思路：向导式 + 一次只做一件事

- **不要**：一屏塞满所有选项、让用户自己选「第几步」。
- **要**：**线性向导**（Wizard），从左到右或从上到下，一步完成再进下一步；每屏只问**一两件必要的事**，其余用默认或智能推断。
- **进度可见**：**顶部进度条仅在前四步显示**（1 基础信息 → 2 文献 → 3 提纲 → 4 预览与支付），展示「第 X / 4 步」，**步骤名可点击回看**（仅查看或回到该步，尽量少乱跳）。**付费并创建订单后，后续步骤（生成中、改稿和降重）不再显示该进度条**，改为订单/任务上下文或独立页。必填项用 **\*** 标出，错误或校验不通过时用**行内提示或 Toast** 明确提示，规范统一。
- **以移动端为主、PC 为辅**：交互与布局**优先保证手机竖屏**（大按钮、少输入、触摸友好、一屏一任务）；PC 端在同一流程下做适配（更宽布局、多栏可选），不单独设计两套流程。

---

## 二、六步在界面上的「傻瓜化」拆解（含预览与支付，生成在后台）

主流程顺序：**1 录入基础信息 → 2 确定参考文献 → 3 确认提纲 → 4 预览与支付 → 5 生成论文（后台执行）→ 6 改稿和降重**。预览与支付放在正式生成前；**生成论文主要在后台执行**。进度条与付费后订单/改稿衔接**见第一节**。支付/确认后进入「生成中」页或订单列表，生成完成后自动跳转订单列表或步骤 6。

### 步骤 1：录入基础信息

**目标**：收集题目、学科、学历/类型、字数、语言、格式等，为后续生成奠定基础。


| 界面元素    | 设计建议                                                                                                                                                                                                                             | 目的                     |
| ------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------- |
| 学科/方向   | **必选**。**点选**（经管 / 计算机 / 人文 / 教育…）：移动端大卡片或底部弹出列表，PC 可用下拉；可加「其他」后填一行字。影响文献推荐与提纲生成，须在本步选定（必填项用 * 标出）。                                                                                                                                                                    | 定领域，影响文献推荐与提纲；移动端避免小下拉           |
| 标题/研究主题 | **必填**。一个短文本框，占位示例：「例如：数字化转型对中小企业成本的影响」。无题目时可点「**智能选题**」按学科+类型推荐若干题目，点选即填入                                                                                                                                                       | 一句话定题；没题目的用户有出路        |
| AI 投喂   | **选填**。研究思路、关键词、核心观点、研究方法等，字数限制如 0/1500；可与开题内容合并为一段输入                                                                                                                                                                            | 提升生成质量，不强制             |
| 开题报告上传  | **选填**。按钮「上传开题报告」，支持 docx/doc；上传后可在输入框核对识别内容再确定                                                                                                                                                                                  | 有开题则生成更贴题              |
| 语言      | **必选**。单选：中文 / 英文                                                                                                                                                                                                                | 与学历、格式一起定写作规范          |
| 学历/类型   | **大按钮/卡片**：专科 / 本科 / 硕士 / 博士 或 本科论文 / 硕士论文 / 期刊稿，自动带出字数范围；移动端单列大块可点                                                                                                                                                              | 不暴露「字数」数字，减少焦虑         |
| 字数      | **必选**。如 4000 字左右、6000 字左右、8000 字左右等，或由学历/类型自动带出                                                                                                                                                                                 | 与 PaperRed 一致，用户可选     |
| 学校/格式   | **可选**：「选择学校或格式模板」—— **按学校全称/简称搜索**（先选学历再搜）；从结果选学校、选「国标/通用」、或选「自定义」；默认「国标/通用」。**未找到该校时**：明确兜底文案「未找到贵校模板，可选国标/通用生成，或通过**联系客服**上传规范要求文档（如学校论文格式规范 PDF/Word），由客服在后台维护并补充该校模板」；入口如右侧「客服与反馈」（或扫码加好友等）。**格式须在本步选定**，步骤 6 改稿与导出均按此处处理。选「自定义」时须在本步内上传 .dotx 或选择参考文献标准，否则按国标/通用。详见文末**附录** | 不同学校格式不同，一步选定；搜索+兜底不卡死 |
| 用户协议    | **必选**。勾选「我已阅读并同意**服务条款与隐私政策**」（或产品实际使用的通用条款名称）。**用户协议放在第一步即可**。与引用、查重相关的知情确认在**步骤 6 导出前**（见步骤 6）。                                                                                                                               | 合规与免责前置                |
| 操作      | 点击「下一步」进入步骤 2                                                                                                                                                                                                                    |                        |


**用户操作**：选学科 + 填标题（或智能选题）+ 选语言 + 选学历/类型 + 选字数 + 可选格式 + 可选 AI 投喂/开题 + 勾选协议 → 点「下一步」。

### 步骤 2：确定参考文献（必选）

**目标**：**必须**选定或导入论文引用的文献，满足类型对应的数量建议；正文引用仅来自本步确定的文献列表，可溯源。**本版本文献为必选，不再支持无料模式**；用户须至少选定系统推荐或上传文献并满足数量建议后才能继续。


| 界面元素   | 设计建议                                                                                               | 目的                      |
| ------ | -------------------------------------------------------------------------------------------------- | ----------------------- |
| 文献来源   | ① **系统推荐**中英文真实文献（按步骤 1 学科/主题）；② **用户粘贴**知网引文格式或上传题录；③ 从知网/万方导入题录（若有接口）。支持「标准引文格式」每行一条，提供格式示例与校验提示 | 用户须从推荐、粘贴、上传中至少一种方式选足文献 |
| 数量建议   | 按学历/类型给出建议：如本科 10～15 篇、硕士 15～25 篇、博士 25 篇以上；底部显示「已选择文献 X 个」；**未满足数量时下一步置灰或提示「请至少选满 X 篇」**          | 用户知道要选多少，且必须选足          |
| 上传区    | 大拖拽区/选择区，上传开题、参考文献 PDF/Word 等；移动端支持文件选择、相册、云盘                                                      | 上传文献的入口                 |
| 文献确认弹窗 | 点击「下一步」前弹出「**文献确认**」：列表展示已选/已导入文献（标题、作者、来源等）；提示「请确认文献是否齐全，格式异常请返回修改」；按钮「返回修改」「确认」                  | 降低选错/漏选就下一步的误操作         |
| 操作     | 选系统推荐和/或粘贴/上传文献 → **满足数量建议** → 在弹窗中确认 → 下一步                                                        |                         |


**用户操作**：选推荐文献和/或粘贴/上传自定义文献 → **满足数量建议** → 在文献确认弹窗中确认 → 点「下一步」。

### 步骤 3：确认提纲

**目标**：确定论文结构，支持编辑、字数控制及图表/公式/代码占位。


| 界面元素       | 设计建议                                                                                           | 目的                    |
| ---------- | ---------------------------------------------------------------------------------------------- | --------------------- |
| 提纲来源       | ① **系统生成**（根据步骤 1～2 的选题、文献）；② **粘贴已有提纲**。不先问「几级大纲」，默认生成后直接展示                                   | 零配置或用户自带              |
| 展示形式       | 树形或缩进列表，每章可展开小节；每行右侧「编辑」小图标，点击可改标题；支持拖拽排序、删除章节。**移动端**：拖拽排序可改为「上移/下移」按钮或长按拖拽，与全文移动端优先一致。                                                 | 默认接受、需要再改；移动端有替代方案             |
| 每章字数       | 每章节字数可微调，范围如 100～2000 字，或由总字数自动分配                                                              | 与 PaperRed 一致，控制篇幅    |
| 图表/公式/代码占位 | 每节右侧可点亮图标：**插入数据(表)、图、公式、代码**；步骤 5 后台生成时在对应位置生成或留空插入                                           | 大纲阶段约定哪里要图表，生成更可控     |
| 操作按钮       | 「就用这个」「微调几处」「重新生成」「粘贴心仪提纲」；「上一步」「下一步」                                                          | 二选一或补充                |
| 确认弹窗       | 标题「确认提纲与插入信息」；展示已插入数据(表)、图、公式、代码的数量；展示「预估生成字数」；提示「AI 将根据提纲生成正文，需要图表公式的请在对应章节点亮图标」；按钮「返回修改」「确定」 | 与 PaperRed 一致，避免误点下一步 |


**用户操作**：看系统生成或粘贴的提纲 → 可选编辑章节、字数、排序，在需要处点亮图表/公式/代码 → 在确认弹窗中核对 → 点「确定」进入步骤 4。

### 步骤 4：预览与支付（正式生成前）

**目标**：让用户确认本次写作信息并完成支付（若产品需要付费），再触发后台生成。


| 界面元素    | 设计建议                                                                          | 目的                 |
| ------- | ----------------------------------------------------------------------------- | ------------------ |
| 预览内容    | 展示**论文标题、提纲摘要（章节目录）、预估字数、已选文献数量**；可选展示部分参考文献列表；正文为「付费/确认后将生成」等锁定提示            | 用户确认再生成，减少误操作      |
| 版本/套餐选择 | 若有商业化：基础版 / AI 率保障版 / 无限改稿版等，展示价格与功能对比；可选赠品（在线改稿、查重、AIGC 检测等）                 | 与 PaperRed 一致，支付前置 |
| 结算与支付   | 显示商品类型（如毕业论文-本科）、总金额、应付金额；「确认并生成」或「扫码支付」等入口；**支付完成后生成订单，同时后台自动开始生成论文**（见步骤 5） | 支付完成即创建订单并触发生成     |
| 操作      | 确认预览信息 → 选择版本（若有）→ 支付或「确认并生成」→ 生成订单并进入步骤 5                                    |                    |


**用户操作**：查看预览 → 选版本（若有）→ 支付或确认 → **生成订单**并进入步骤 5（生成论文）。若无需支付，则「确认并生成」后直接生成订单并进入步骤 5。

### 步骤 5：生成论文（后台执行，与订单合并）

**目标**：**支付完成后生成订单，同时后台自动生成论文**；本步与订单创建一体，**本步及步骤 6 均不再显示顶部进度条**。**订单列表**即用户查看生成进度与进入改稿的入口；用户进入「生成中」页或订单列表查看状态，生成完成后在订单列表进行下载或进入在线改稿。


| 界面元素              | 设计建议                                                                                    | 目的                  |
| ----------------- | --------------------------------------------------------------------------------------- | ------------------- |
| 与订单、进度条的关系        | **支付完成即创建订单，并同时启动后台按章生成论文**。付费后进度条不再显示，本步与步骤 6 在订单列表或改稿页完成。 | 订单与生成合一；进度条见第一节    |
| 订单列表（条目与操作）       | **每条订单展示**：标题、创建时间、**论文生成状态**（如生成中 / 已生成第 X 章 / 写作完成 / 失败）。生成中可点进查看「生成中」页详细进度；失败可重试或联系客服。**操作**：**查看**、**下载**、**在线改稿**。**仅状态为「写作完成」的订单可进入在线改稿**；未完成时「在线改稿」置灰或隐藏，完成后再开放。从列表点击「在线改稿」即进入步骤 6 改稿页。 | 订单条目展示状态；订单即改稿入口，仅完成可进 |
| 后台行为              | 系统按步骤 1～3 的基础信息、文献、提纲**在后台按章依次生成全文**；正文引用仅来自步骤 2 文献列表；每章生成后可做引用溯源校验（结果在步骤 6 或改稿时展示）     | 用户无需在前台逐章操作，体验更轻    |
| 前端展示（生成中页）        | 「生成中」页可展示状态文案与进度条/百分比；用户可离开页面，在**订单列表**查看同一订单状态。 | 状态在订单上可见，不丢进度       |
| 完成跳转              | 生成完成后：订单状态更新为「写作完成」，操作显示「下载」「在线改稿」；可自动跳转订单列表或改稿页。                                        | 与订单列表、在线改稿衔接        |
| 引用约束              | 后台生成时 prompt 明确「正文引用仅来自步骤 2 确定的 ref 列表」；生成后可做全文引用溯源校验，不可溯源处在步骤 6 或在线改稿时标出               | 质量可控，防控幻觉           |


**用户操作**：支付/确认后生成订单并进入「生成中」页或订单列表 → 等待（可离开，在订单列表查看该订单的生成状态）→ 生成完成后在订单列表点击「下载」或「在线改稿」。

### 步骤 6：改稿和降重（参考 PaperRed 智能论文助手）

**目标**：在**改稿页**完成加图/表、按意见修改、降重等，系统执行引用核对与查重后，按步骤 1 格式**保存下载**。布局参考 PaperRed：**左侧正文、右侧能力区**，重点突出**加图片和表格**、**按意见修改**、**降重**。


| 界面元素             | 设计建议                                                                                                                                              | 目的                               |
| ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------- |
| **改稿布局**         | **左侧**：论文正文（富文本，按章可折叠）；**右侧**（或移动端底部/抽屉）：能力区（详见本表下列能力项）。**入口**：从订单列表点击「在线改稿」进入（仅状态为「写作完成」的订单可进，见步骤 5 订单列表）。                                                                  | 与 PaperRed 智能论文助手一致，一屏完成改稿       |
| **加图片和表格**（内容插入） | **重点能力**。入口如「内容插入」：**插图**（需结合的内容 + 提示词，提交生成后插入）、**插表**（表格数据/文本 + 格式要求，提交生成后插入）；可选公式、代码。弹窗填写后「提交生成」，生成完成插入到正文                                      | 满足导师/学校对图表的要求，与 PaperRed 文档编辑区一致 |
| **按意见修改**（修文）    | **重点能力**。入口如「意见修改」或「修文」：输入**修改意见/导师建议**（或选中段落）+ 提示词（学术风格、表达方式等）+ 预期字数 → 提交生成，AI 按条修改正文（仅针对文本，不修改图/表/公式/代码）                                        | 根据导师/审稿意见逐条改，与 PaperRed 意见修改一致   |
| **降重**           | **重点能力**。入口如「文本优化」下的「降重」：对选中段落或全文执行降重，降低重复率/查重率；可配合**查重**结果针对性修改。可选「润色、扩写、缩写、降 AI 痕迹」等同一分组                                                        | 重复率达标，与 PaperRed 文本优化一致          |
| 引用核对             | 系统调用第三方引用核对接口（或自建与步骤 2 文献列表比对），自动/一键执行；结果展示「可溯源 X 处、不可溯源 Y 处」及明细，用户可在改稿中修改。用户无需手动逐条核对                                                             | 引用由系统核对，改稿中修正                    |
| 查重               | **接入第三方查重 API 或使用自定义查重模块 API**，传入正文或导出用 Word 获取重复率与报告并展示；用户可先查重再在改稿页用「降重」针对性改。                                                                    | 查重结果驱动降重                         |
| 格式检查与导出          | **按步骤 1 所选学校/格式模板**检查标题层级、图表编号、参考文献格式。**导出前确认**：用户须确认「已查看引用核对结果」「已查看查重结果」后再允许**保存下载**（与引用、查重相关的知情确认放在本步，不在步骤 1）。**主按钮「保存下载」**（保存当前编辑并下载 Word/PDF） | 改稿结果持久化并符合格式；引用与查重知情在导出前完成       |


**用户操作**：进入步骤 6 改稿页 → 使用**加图片和表格**、**按意见修改**、**降重**等能力改稿 → 触发或查看引用核对与查重结果 → **确认已查看引用与查重结果**后点击「**保存下载**」。**无需用户手动查重或逐条核对引用**。

## 三、全局「越简单越好」的 11 条原则

1. **一屏一任务**：每屏只完成一个步骤的一小件事，不堆表单。
2. **默认优先**：能推断的都默认选好（字数、格式、大纲层级），高级项折叠或放「设置」。
3. **主流程零必填**：除「选题」那一句主题外，**步骤 2 文献为必选**（须上传或选择文献并满足数量建议），其余尽量都可跳过或用默认（如不微调大纲）。**必填项用 \*** 标出，**错误提示**统一（行内红色或 Toast），不散落多种样式。
4. **主按钮唯一**：每屏一个主按钮（如「下一步」「确认并生成」「保存下载」），颜色/面积突出。
5. **进度可回看不可乱跳**：**顶部进度条仅在前四步显示**（「第 X/4 步」可点击回看）；付费创建订单后不再显示进度条。可回到上一步查看或改，但不鼓励从步骤 6 直接跳到步骤 2，避免状态错乱。
6. **文案说人话**：不用「文献综述」「RACCCA」等术语；用「定题目」「选文献」「定提纲」「改稿降重并下载」。
7. **等待时有反馈**：生成大纲、章节、润色时用加载动画 + 短文案（如「正在生成第三章…」），超过 5 秒可加进度或「预计还需 X 秒」。
8. **错误可恢复**：生成不满意可在**在线改稿**中修文或重新生成某段；误删用「撤销」或自动保存历史版本（可选）。
9. **以移动端为主、同时支持 PC 端**：产品**以移动端为主**，PC 端同步支持。移动端：**竖屏单栏**、**大按钮**（触摸热区不小于 44pt）、**少输入**（多用手选/点选）、**一屏一任务**，主流程（基础信息→文献→提纲→预览与支付→生成论文→改稿和降重并导出）全部可在手机完成；步骤 5 生成在后台，前端只显示状态与进度；**在线改稿**在移动端可采用**简化富文本**（如固定字号、精简工具栏）或分段编辑，保证可用。PC 端：同一流程，布局适配宽屏（如改稿页左侧正文+右侧能力区），不要求用户「必须用电脑」。两端数据与进度互通，用户可随时换设备继续。
10. **首次使用有引导**：第一次进入可 3～5 屏的简短「引导」（每屏一句话 + 图），说明「填基础信息 → 选文献 → 定提纲 → 预览与支付 → 生成论文 → 改稿和降重并下载」，然后进入步骤 1。
11. **订单/任务可见**：若有订单或写作任务，**列表展示状态**（生成中/已完成/失败）与**操作**（查看、下载、在线改稿），用户不丢稿、可随时改稿或下载。

## 四、技术实现上的「简单」建议

- **前端与多端**：**以移动端为主**，同时支持 PC。采用**响应式布局或移动优先**（如先做移动版再 media query 适配 PC，或使用多端统一框架）；现成 UI 库选用**支持移动端**的 Steps/Wizard + 表单 + 富文本（如 TipTap、Quill 在移动端可配置简化工具栏、大触摸区），减少自绘组件。步骤 1 的学科/类型在移动端宜用**大卡片或底部 sheet 点选**，避免小下拉框。
- **状态**：把「当前步骤、选题、大纲、每章内容、润色结果」放在一个全局状态（如 Zustand/Redux）或服务端会话里，便于「上一步/下一步」和刷新不丢。
- **AI 调用**：步骤 1～3 的基础信息、文献、提纲与**步骤 5 后台按章生成**统一走后端，把「选题+文献列表+提纲」及按章上下文打包成 prompt，且 **prompt 中明确约束「正文引用仅来自步骤 2 确定的 ref 列表」**，避免前端拼长文。**步骤 6 改稿和降重**中的加图/表、按意见修文、降重等能力单独走对应接口（插图/插表/修文/降重等）。
- **引用溯源校验**：步骤 5 后台每章生成后可做引用与步骤 2 的 ref 列表匹配，或将全文与 ref 列表提交**第三方引用核对接口**，将无法对应的引用标出，在步骤 6 改稿页中展示，供用户修改或删除；可做成独立接口或嵌入生成完成后的流水线。技术规格阶段可补充**匹配策略**：区分「确定无法溯源」与「可能匹配请确认」，减少误报。
- **查重与引用核对的对接**：步骤 6 改稿与导出前或改稿页内，**接入第三方查重 API 或使用自定义查重模块 API**（传入正文或导出用 Word）获取重复率与报告；**调用第三方引用核对接口**（若有）或沿用自建「引用与 ref 列表比对」逻辑。结果统一展示给用户，用户无需手动查重或逐条核对。不考虑查重降级方案。
- **格式与导出**：不同大学对论文格式要求不同，需支持**按学校/格式模板导出**。实现上：维护「学校/格式 ID → 模板」映射（含页眉页脚、标题层级、正文样式、参考文献著录格式，如 GB/T 7714）；用 docx 库或服务端模板根据**用户所选学校/格式 + 论文类型**生成 Word。若用户选「自定义」，须在步骤 1 内上传 .dotx 或选择参考文献标准（国标/APA 等），不推迟到导出前，以便步骤 6 格式检查有依据。

## 五、每一步：质量可控要点（与第二节对应）

流程顺序与第二节一致：**1 录入基础信息 → 2 确定参考文献 → 3 确认提纲 → 4 预览与支付 → 5 生成论文（后台，不占进度条）→ 6 改稿和降重**。以下仅保留与**质量可控**直接相关的分析，界面与操作细节见第二节。

---

### 步骤 1：录入基础信息

**质量关联**：基础信息越完整，后续文献推荐、提纲与正文越可控；格式在本步选定后步骤 6 改稿与导出才有统一依据。协议勾选为合规与免责前置（本步仅通用条款；引用与查重知情在步骤 6 导出前）。

---

### 步骤 2：确定参考文献

**说明**：本版本文献为必选，**不再支持无料模式**；用户须至少选定系统推荐或上传文献并满足数量建议后才能继续。

**质量关联**：系统约束「正文引用仅来自本步确定的文献列表」并做溯源校验，是防控幻觉的关键；将已选文献作为步骤 5 唯一引用来源。

---

### 步骤 3：确认提纲

**质量关联**：提纲是全文骨架；用户确认后步骤 5 按章生成才有统一结构；占位约定哪里要图表，避免事后补。

---

### 步骤 4：预览与支付

**质量关联**：预览与支付前置，用户知情后再生成；订单/任务与步骤 5 状态衔接。

---

### 步骤 5：生成论文（后台执行）

**质量关联**：后台生成 + 引用约束 + 完成后溯源校验，保证正文质量可控；用户仅在「生成中」页等待或离开至订单列表查看。

---

### 步骤 6：改稿和降重

**质量关联**：改稿页集中「加图/表、按意见改、降重」，引用核对与查重由系统完成，用户须在**导出前确认已查看引用核对结果与查重结果**后再保存下载；既保证稿子可改、可降重、可核对，又避免用户多端跳转。

---

## 六、小结：为保质量不能省的用户动作与信息


| 步骤      | 不能省的用户提交信息                                             | 不能省的用户交互动作                                  |
| ------- | ------------------------------------------------------ | ------------------------------------------- |
| 1 基础信息  | 学科、标题、语言、学历/类型、字数；可选：学校/格式；必选：协议（服务条款与隐私政策等通用条款）       | 选择 + 输入（或智能选题）+ 选择 + 勾选协议 + 下一步             |
| 2 文献    | **必选**：推荐+上传，须满足数量建议，文献确认弹窗                            | 选文献/上传 → 满足数量 + 确认弹窗 + 下一步                  |
| 3 提纲    | 对提纲的采用或微调；可选每章字数、图表公式代码占位                              | 查看 + 确认/微调 + 确认弹窗 + 确定                      |
| 4 预览与支付 | 确认预览信息；若有商业化则选版本并支付                                    | 查看预览 → 支付或「确认并生成」→ 进入步骤 5                   |
| 5 生成论文  | 无（后台执行）；用户可见状态与进度                                      | 等待（可离开至订单列表查看）→ 完成后查看/下载/在线改稿               |
| 6 改稿和降重 | 加图片和表格、按意见修改、降重；引用核对与查重（系统执行）；**导出前确认已查看引用与查重结果**；保存下载 | 进入改稿页 → 加图/表、按意见修文、降重 → 确认已查看引用/查重结果 → 保存下载 |


界面可以简化文案、减少分叉、用默认值减少选择，但**上述「需提交信息」和「必做动作」若被自动跳过或替代，论文质量的可控性会下降**。设计时应在「简单」与「必要输入/确认」之间取得平衡。

## 附录：格式模板与不同学校/单位要求

不同大学、院系对毕业论文的**版式与格式**要求差异很大；格式须在步骤 1 选定、步骤 6 改稿与导出均按此处处理（见第二节步骤 1、步骤 6）。以下为常见差异项与产品建议。

**常见差异项与系统支持方式**：


| 常见差异项        | 说明                                               | 系统可支持方式                                    |
| ------------ | ------------------------------------------------ | ------------------------------------------ |
| **参考文献著录规范** | 国标 GB/T 7714、APA、MLA、各校自定顺序与标点等                  | 预设「国标/通用」+ 按学校选择（学校模板内绑定该规范）；或单独选项「参考文献格式」 |
| **标题层级与编号**  | 一级/二级/三级标题的字体、字号、编号样式（如 1 / 1.1 / 1.1.1 或 一、（一）） | 每个学校模板内预定义；导出时按模板套用                        |
| **页眉页脚**     | 页眉是否写学校名、论文题目、页码位置与样式                            | 学校模板内预定义                                   |
| **正文与版心**    | 字体、字号、行距、段前段后、页边距                                | 学校模板内预定义                                   |
| **摘要与目录**    | 中英文摘要格式、目录是否自动生成、是否单独页                           | 学校模板内预定义                                   |


**产品设计建议**：

- **步骤 1 选定格式**：按学校全称/简称搜索（先选学历再搜）、「国标/通用」、「自定义」；未找到该校时兜底文案与入口见第二节步骤 1。选「自定义」须在本步内上传 .dotx 或选择参考文献标准，否则步骤 6 按「国标/通用」执行。
- **模板维护**：维护「学校 ID → 格式模板」映射；可优先覆盖常见高校，其余用「国标/通用」。用户未找到学校时，可通过联系客服（如客服联系与反馈、扫码加好友）**上传规范要求文档**（如学校论文格式规范 PDF/Word），由客服/运营在**后台维护**并补充该校模板后上线。
- **改稿与导出**：步骤 6 格式检查与保存下载均读取步骤 1 已选学校/格式（见第二节步骤 6）。
