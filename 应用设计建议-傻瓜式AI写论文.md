# 傻瓜式「AI 写论文」应用设计建议

目标：**界面清晰、步骤最少、操作极简**，让用户像填表一样完成论文写作引导。**以移动端为主、同时支持 PC 端**，主流程在手机竖屏下即可完成。

---

## 一、整体思路：向导式 + 一次只做一件事

- **不要**：一屏塞满所有选项、让用户自己选「第几步」。
- **要**：**线性向导**（Wizard），从左到右或从上到下，一步完成再进下一步；每屏只问**一两件必要的事**，其余用默认或智能推断。
- **进度可见**：顶部或左侧始终显示「第 X / 6 步」，可点步骤名回看（仅查看，尽量少让用户回头改）。
- **以移动端为主、PC 为辅**：交互与布局**优先保证手机竖屏**（大按钮、少输入、触摸友好、一屏一任务）；PC 端在同一流程下做适配（更宽布局、多栏可选），不单独设计两套流程。

---

## 二、六步在界面上的「傻瓜化」拆解

### 步骤 1：选题（用户只填 3 个框）

| 界面元素 | 设计建议 | 目的 |
|----------|----------|------|
| 学科/方向 | **点选**（经管 / 计算机 / 人文 / 教育…）：移动端用**大卡片或底部弹出列表**，PC 可用下拉；可加「其他」后填一行字 | 不让用户想「该写啥」；移动端避免小下拉 |
| 研究主题 | **一个短文本框**，占位符示例：「例如：数字化转型对中小企业成本的影响」 | 一句话定题，不写长段 |
| 字数/类型 | **大按钮/卡片**：本科论文 / 硕士论文 / 期刊稿，自动带出字数范围；移动端单列大块可点，便于竖屏触摸 | 不暴露「字数」数字，减少焦虑；移动端友好 |
| 学校/格式 | **可选**：「选择学校或格式模板」—— 从列表选**学校**（系统预设该校常用版式与参考文献规范）、选「国标/通用」、或选「自定义」；默认「国标/通用」。**格式须在步骤 1 选定**，步骤 5 润色与步骤 6 导出均按此处所选处理。选「自定义」时，**须在本步内**上传单位 .dotx 或选择参考文献标准（如 GB/T 7714），否则步骤 5、6 按国标/通用执行。详见文末**附录：格式模板说明**。 | 不同大学对版式、参考文献等要求不同，一步选定后润色与导出统一依据 |

**用户操作**：选 1 个 + 填 1 句 + 点 1 个 + 可选选学校/格式 → 点「下一步」。

---

### 步骤 2：大纲（用户只做「确认/微调」）

| 界面元素 | 设计建议 | 目的 |
|----------|----------|------|
| 生成方式 | **不先问**「你要几级大纲」；系统根据步骤 1 的选题+类型，**自动生成一份默认大纲**直接展示 | 零配置 |
| 展示形式 | 树形或缩进列表，每章可**展开看小节**；每行右侧有「编辑」小图标，点击才弹出简短输入框 | 默认接受、需要再改 |
| 操作 | 两个主按钮：「就用这个」 / 「微调几处」；微调时只改标题文字，不要求用户自己加章节 | 二选一，无复杂操作 |

**用户操作**：看一遍 → 可选改几处标题 → 点「下一步」。

---

### 步骤 3：文献与资料（区分「有料」与「无料」模式）

| 界面元素 | 设计建议 | 目的 |
|----------|----------|------|
| 上传区 | **一个大拖拽区/选择区**：「上传开题报告、课程作业或参考文献 PDF/Word，可多文件」；**移动端**支持系统文件选择、相册、云盘入口，避免依赖拖拽 | 有料模式的核心入口；移动端以点选为主 |
| 模式区分 | **有料模式**：用户上传了文献/资料 → 生成时引用可溯源，质量可控；系统在后续生成与导出中仅允许引用上传列表中的文献。 | 引用有据可查，降低幻觉 |
| | **无料模式**：用户选择「不上传，直接下一步」→ 系统**明确提示**：「当前未上传文献，生成内容仅供参考，文内引用需您自行核实；导出文件将带有说明。」导出件（Word/PDF）**增加水印或文末声明**（如「本文由 AI 辅助生成，未绑定用户上传文献，引用与数据请自行核对」）。 | 避免用户误以为不传也能得到可信论文 |
| 可选 | 若有「从知网/万方导入题录」的接口，做成**一个按钮 + 可选粘贴网址或 DOI**，解析后自动当「指定文献」 | 进阶功能，默认折叠 |

**用户操作**：上传 1～N 个文件（有料）或点「不上传，直接下一步」（无料，并确认看到上述提示）→ 点「下一步」。

---

### 步骤 4：写正文（按章推进，一次一章）

| 界面元素 | 设计建议 | 目的 |
|----------|----------|------|
| 章节选择 | 左侧或顶部：**步骤 2 里定好的章节目录**，当前章高亮，其余灰或可点 | 心里有数「写到哪了」 |
| 当前章内容 | 中间大块：**当前章 AI 生成的初稿**，可编辑的富文本（或 Markdown）；顶部一个「重新生成本章」按钮 | 不一次生成全文，避免卡顿和混乱 |
| 生成策略 | **默认推荐「按章生成」**：用户点进某章 → 自动带出上一章摘要/大纲上下文 → 点「生成本章」→ 查看/修改后再进下一章；保证逐章确认，质量可控。 | 用户只点「生成」和「改字」 |
| 引用约束与幻觉防控 | **正文中所有引用必须来自用户上传的 ref 列表**（步骤 3 有料模式下的上传文献，或第八节场景下的 10 篇 ref）。系统在**每章生成后做引用溯源校验**：可**自建逻辑**（解析正文引用与 ref 列表比对）或**调用第三方引用核对接口**，**标出无法对应到该列表的引用**（如「未在您上传的文献中找到对应来源」），提示用户修改或删除，避免 AI 捏造第 11、12 篇文献或不存在的数据。 | 质量可控的核心，防控幻觉；可对接第三方实现 |
| 一键生成全部（可选） | 若提供「一键生成全部章节」：作为**快速模式**，须在 UI 上**明确警示**，如「一键模式将跳过逐章确认，质量可控性降低，建议有时间时使用按章生成」。用户知情后可选。 | 与「逐章确认」立场统一，不误导 |
| 提示 | 每章下方可放**一句短提示**（如：「文献综述请突出近 5 年与本研究空白」），用系统预设，用户可不看 | 懂的人能参考，不懂的人不碍事 |

**用户操作**：选一章 → 点「生成本章」→ 看/改内容（并处理系统标出的不可溯源引用）→ 再选下一章重复；全部写完点「下一步」。若选一键生成全部，需先确认看到上述警示。

---

### 步骤 5：润色与格式（一键 + 可选）

| 界面元素 | 设计建议 | 目的 |
|----------|----------|------|
| 主操作 | **一个大按钮**：「一键学术润色 + 检查格式」；背后做：语气学术化、术语统一、**按用户在步骤 1 所选学校/格式模板**检查标题层级、图表编号、参考文献格式等 | 一个动作解决大部分问题，且符合目标学校要求 |
| 可选 | 若支持「降重/去 AI 味」：做成**勾选项**「同时降低 AI 痕迹（语义改写）」默认勾选；高级选项折叠 | 默认就做，专家可关 |
| 结果 | 润色后直接在当前文档上替换或生成「润色版」副本，并提示「已生成润色版，请在步骤 6 核对」 | 不增加「选哪一版」的困惑 |

**用户操作**：点「一键润色」（可选勾选降重）→ 点「下一步」。

---

### 步骤 6：核查与导出（系统调用第三方核对引用 + 查重，无需用户手动操作）

| 界面元素 | 设计建议 | 目的 |
|----------|----------|------|
| 引用核对 | **由系统调用第三方引用核对接口**（或自建引用与 ref 列表比对），在进入步骤 6 或导出前自动/一键执行；结果展示给用户：如「可溯源引用 X 处、不可溯源 Y 处」及明细，用户可据此回到步骤 4 修改。**用户无需手动逐条核对**，只需查看系统返回结果并确认或修改。 | 引用核对由系统完成，减轻用户负担 |
| 查重 | **由系统调用第三方查重 API**，在导出前自动/一键发起查重；展示**重复率与简要报告**（如总重复率、各段重复来源），用户查看后即可下载。**用户无需自行复制到学校查重系统或手动查重**。若第三方有次数或费用限制，可在 UI 注明「本次查重已使用 1 次」等。 | 查重由系统完成，无需用户手动查重 |
| 核查清单 | **勾选列表**：在系统已执行引用核对与查重并展示结果的前提下，用户**确认「已查看引用核对结果」「已查看查重结果」**后即可下载；其余项（格式已检查、数据已核对等）可默认勾好或由用户自选。不要求用户「主动勾选我已手动核对/查重」。 | 清单体现为「已查看系统结果」，不依赖用户手动操作 |
| 导出 | **一个主按钮**：「下载论文（Word / PDF）」；**版式与格式按步骤 1 所选学校/格式模板**；在系统完成引用核对与查重并展示结果、用户确认已查看后可点击 | 不同学校一次选好，导出即符合该校要求 |

**用户操作**：进入步骤 6 → 系统自动/一键执行**引用核对**与**查重**（调用第三方接口）→ 用户**查看引用核对结果与查重报告**（重复率等）→ 确认已查看后 → 下载论文。**无需用户手动查重或逐条核对引用**。

## 三、全局「越简单越好」的 10 条原则

1. **一屏一任务**：每屏只完成一个步骤的一小件事，不堆表单。
2. **默认优先**：能推断的都默认选好（字数、格式、大纲层级），高级项折叠或放「设置」。
3. **主流程零必填**：除「选题」那一句主题外，其余尽量都可跳过或用默认（如不微调大纲）；步骤 3 不传文献时走「无料模式」，须明确提示风险并在导出时加声明，不淡化引用需自行核实的责任。
4. **主按钮唯一**：每屏一个主按钮（如「下一步」「生成本章」「一键润色」「下载」），颜色/面积突出。
5. **进度可回看不可乱跳**：可回到上一步查看或改，但不鼓励从步骤 6 直接跳到步骤 2，避免状态错乱。
6. **文案说人话**：不用「文献综述」「RACCCA」等术语；用「定题目」「列目录」「上传你的资料」「写每一章」「改得更像论文」「最后检查并下载」。
7. **等待时有反馈**：生成大纲、章节、润色时用加载动画 + 短文案（如「正在生成第三章…」），超过 5 秒可加进度或「预计还需 X 秒」。
8. **错误可恢复**：生成不满意就「重新生成本章」；误删用「撤销」或自动保存历史版本（可选）。
9. **以移动端为主、同时支持 PC 端**：产品**以移动端为主**，PC 端同步支持。移动端：**竖屏单栏**、**大按钮**（触摸热区不小于 44pt）、**少输入**（多用手选/点选）、**一屏一任务**，主流程（选题→大纲→资料→按章写→润色→导出）全部可在手机完成；步骤 4 按章编辑在移动端可采用**简化富文本**（如固定字号、精简工具栏）或分段编辑，保证可用。PC 端：同一流程，布局适配宽屏（如步骤 4 左侧目录+右侧正文双栏），不要求用户「必须用电脑」。两端数据与进度互通，用户可随时换设备继续。
10. **首次使用有引导**：第一次进入可 3～5 屏的简短「引导」（每屏一句话 + 图），说明「选题目 → 定目录 → 上传资料 → 按章写 → 润色 → 下载」，然后进入步骤 1。

## 四、技术实现上的「简单」建议

- **前端与多端**：**以移动端为主**，同时支持 PC。采用**响应式布局或移动优先**（如先做移动版再 media query 适配 PC，或使用多端统一框架）；现成 UI 库选用**支持移动端**的 Steps/Wizard + 表单 + 富文本（如 TipTap、Quill 在移动端可配置简化工具栏、大触摸区），减少自绘组件。步骤 1 的学科/类型在移动端宜用**大卡片或底部 sheet 点选**，避免小下拉框。
- **状态**：把「当前步骤、选题、大纲、每章内容、润色结果」放在一个全局状态（如 Zustand/Redux）或服务端会话里，便于「上一步/下一步」和刷新不丢。
- **AI 调用**：步骤 2/4/5 的生成与润色统一走后端，把「选题+类型+大纲+上传材料+当前章上下文」打包成 prompt，且 **prompt 中明确约束「正文引用仅来自用户上传/锁定的 ref 列表」**，避免前端拼长文。
- **引用溯源校验**：生成每章后，后端解析正文中的引用与本次任务的上传/锁定 ref 列表做匹配，或将正文与 ref 列表提交给**第三方引用核对接口**，将无法对应的引用标出并返回前端展示，供用户修改或删除；可做成独立接口或嵌入生成接口的后续步骤。技术规格阶段可补充**匹配策略**：区分「确定无法溯源」（与 ref 列表无匹配）与「可能匹配请确认」（模糊匹配需用户确认），减少误报。
- **查重与引用核对的第三方对接**：步骤 6 导出前，**调用第三方查重 API**（传入正文或导出用 Word）获取重复率与报告；**调用第三方引用核对接口**（若有）或沿用自建「引用与 ref 列表比对」逻辑。结果统一展示给用户，用户无需手动查重或逐条核对。**查重 API 降级**：国内主流查重（知网、维普等）多无公开 API、接入成本高且有版权限制。若**未接入第三方查重 API**，步骤 6 降级为：提供**下载 Word** + **引导用户前往学校/指定查重系统**（如文案「请下载后使用学校提供的查重系统检测」），并在 UI 上简要说明原因（如「暂未接入查重服务，请下载论文后自行查重」），避免因无 API 导致步骤 6 无法完成。
- **格式与导出**：不同大学对论文格式要求不同，需支持**按学校/格式模板导出**。实现上：维护「学校/格式 ID → 模板」映射（含页眉页脚、标题层级、正文样式、参考文献著录格式，如 GB/T 7714）；用 docx 库或服务端模板根据**用户所选学校/格式 + 论文类型**生成 Word；无料模式导出时注入水印或文末声明。若用户选「自定义」，须在步骤 1 内上传 .dotx 或选择参考文献标准（国标/APA 等），不推迟到导出前，以便步骤 5 格式检查有依据。

## 五、可选的「更傻瓜」增强

- **模板选题**：步骤 1 除自定义主题外，提供「从热门题目里选一个」（如经管 10 题、计算机 10 题），点选即填好主题与类型。
- **智能续写**：在步骤 4 每章底部加「续写这一段」小按钮，选中一段话即可让 AI 只扩写这一段，减少「整章重写」的负担。
- **一句式总结**：步骤 6 顶部用一两句话总结「您的论文：题目是 XX，共 X 章，已润色，请按清单核对后下载」，强化完成感。
- **移动端优先的进一步优化**：步骤 6 导出在移动端提供「保存到手机/文件」或「分享到微信/邮件」；步骤 4 长文在移动端可提供「折叠目录 + 上滑加载」以减轻卡顿；两端登录与进度同步，便于手机与电脑切换使用。

## 六、每一步：用户交互动作与需提交信息（质量可控分析）

以下按步骤列出：**用户需要做的交互动作**、**用户需要提交/确认的信息**。保证这些输入到位，论文质量才可控；界面简化时不应省略或自动替代这些环节。

---

### 步骤 1：选题

| 类型 | 内容 | 说明 |
|------|------|------|
| **用户需提交的信息** | **学科/方向**（必填） | 决定论文领域、术语和文献范围；缺失则生成内容易泛化或跑题。 |
| | **研究主题/题目**（必填，一句话） | 决定全文核心问题和写作重心；越具体，生成越贴题。 |
| | **字数或论文类型**（必选） | 本科/硕士/期刊等，决定篇幅、深度和结构；影响大纲与每章长度。 |
| | **学校或格式模板**（可选） | 不同大学对版式、参考文献等要求不同；**须在步骤 1 选定**，步骤 5、6 均按此处理。选「自定义」时须在本步内上传 .dotx 或选参考文献标准，否则按国标/通用；未选则用国标/通用默认。详见附录。 |
| **用户交互动作** | 选择学科（下拉或点选） | 一次选择。 |
| | 输入或粘贴研究主题（短文本） | 可提供示例占位，减少空白焦虑。 |
| | 选择论文类型（点选预设按钮） | 一次选择。 |
| | 可选：选择学校或格式模板（列表选学校 / 国标通用 / 自定义；选自定义时本步内上传 .dotx 或选参考文献标准） | 格式在步骤 1 确定，润色与导出统一依据。 |
| | 点击「下一步」 | 进入步骤 2；未填主题时宜置灰或提示。 |

**质量关联**：选题越明确，后续大纲、章节、润色越可控；不建议用「推荐题一点即过」替代用户主动输入主题，除非用户明确点选某推荐题（等同提交了该主题）。

---

### 步骤 2：大纲

| 类型 | 内容 | 说明 |
|------|------|------|
| **用户需提交/确认的信息** | **对生成大纲的确认或修改** | 至少「采用」或「微调」；若微调，需提交修改后的章节标题（可逐条编辑）。 |
| **用户交互动作** | 查看系统生成的大纲 | 阅读树形/缩进列表，可展开小节。 |
| | 二选一：「就用这个」或「微调几处」 | 主路径清晰；选微调则进入编辑。 |
| | 若微调：点击某行「编辑」→ 修改标题文字 → 确认 | 不要求用户自己增删章节，只改标题。 |
| | 点击「下一步」 | 将确定后的大纲作为步骤 4 的写作依据。 |

**质量关联**：大纲是全文骨架；用户确认或微调后，后续按章生成才有统一结构。不建议「自动生成后不经过用户确认直接进入写正文」，否则结构偏差难以补救。

---

### 步骤 3：文献与资料

| 类型 | 内容 | 说明 |
|------|------|------|
| **用户需提交的信息** | **有料模式**：上传文件（开题报告、课程论文、参考文献 PDF/Word 等）或指定文献（知网/万方/DOI） | 有则引用可溯源，生成仅允许引用该列表，质量可控。 |
| | **无料模式**：明确选择「不上传，直接下一步」 | 系统须明确提示：生成内容仅供参考，引用需自行核实；导出件加水印或文末声明。 |
| **用户交互动作** | 选择：上传文件（有料）或「不上传，直接下一步」（无料） | 无料时须让用户看到风险提示后再进入下一步。 |
| | 若上传：确认文件列表（可删除误传） | 避免传错文件。 |
| | 点击「下一步」 | 将已上传内容作为步骤 4 的**唯一允许引用来源**（有料时）。 |

**质量关联**：有料模式下系统约束「正文引用仅来自上传列表」并做溯源校验，是防控幻觉的关键；无料模式必须明确告知「引用需自行核实」并在导出时加声明，避免用户误以为不传文献也能得到可信论文。

---

### 步骤 4：写正文

| 类型 | 内容 | 说明 |
|------|------|------|
| **用户需提交/确认的信息** | **按章生成请求** | 用户进入某章后点击「生成本章」，即提交「要写这一章」的意图；系统用选题+大纲+上文+**仅限用户上传/锁定的 ref 列表**生成。 |
| | **对每章内容的审阅与修改** | 可编辑初稿（增删改）、可「重新生成本章」；修改结果即用户确认后的正文。 |
| **系统必做（幻觉防控）** | **引用溯源校验** | 每章生成后，系统解析正文中的引用（如 [1]、作者年份），与用户上传/锁定的文献列表比对，**标出无法对应到该列表的引用**，提示用户修改或删除，再允许进入下一章或下一步。正文中所有引用必须来自该列表，不得出现「第 11、12 篇」等未上传文献。 |
| **用户交互动作** | 在章节目录中选择某一章 | 明确当前要写/改的章节。 |
| | 点击「生成本章」 | 触发该章生成；等待时显示进度。 |
| | 阅读生成内容，**处理系统标出的不可溯源引用**；可选：直接采纳 / 编辑修改 / 重新生成 | 保证每章都经过用户过一次目或手，且无未溯源引用。 |
| | 对全部章节都至少生成并过目后，点击「下一步」 | 可校验「是否每章都有内容」再放行。 |
| **一键生成全部（若提供）** | 作为快速模式，UI 上须**明确警示**：「一键模式跳过逐章确认，质量可控性降低」；用户知情后可选。生成完成后仍须做**全文引用溯源校验**并标出不可溯源引用，由用户统一处理。 | 与「逐章确认」立场统一，不误导。 |

**质量关联**：按章生成 + 用户逐章确认 + **引用溯源校验**，是质量可控的核心；不建议全自动连写各章且不经过用户逐章确认就进入润色。若提供一键生成全部，须加警示并保留生成后的引用校验。

---

### 步骤 5：润色与格式

| 类型 | 内容 | 说明 |
|------|------|------|
| **用户需提交/确认的信息** | **是否执行润色** | 用户点击「一键润色」即确认执行。 |
| | **是否同时降重/去 AI 味** | 勾选项，默认可勾选；用户不勾则只做学术化与格式，不做语义改写。 |
| **用户交互动作** | 点击「一键学术润色 + 检查格式」 | 触发润色；等待时显示进度。 |
| | 可选：勾选/取消「同时降低 AI 痕迹」 | 保留用户选择权，避免误伤需要保留的表述。 |
| | 查看润色结果（可选对比原文） | 确认后再进下一步。 |
| | 点击「下一步」 | 进入核查与导出。 |

**质量关联**：润色和降重是两件事；降重会改语义，用户应知情并可选。不建议默认强制降重且不提供关闭，否则可能破坏用户刻意保留的表述。

---

### 步骤 6：核查与导出

| 类型 | 内容 | 说明 |
|------|------|------|
| **系统必做** | **调用第三方引用核对接口**（或自建引用与 ref 列表比对） | 在步骤 6 自动/一键执行，将「可溯源/不可溯源」结果展示给用户；用户无需手动逐条核对。 |
| | **调用第三方查重 API** | 在导出前自动/一键发起查重，展示重复率与简要报告；**用户无需手动查重**。 |
| **用户需提交/确认的信息** | **查看并确认系统返回的引用核对结果与查重结果** | 用户确认「已查看引用核对结果」「已查看查重结果」后即可下载；不要求用户主动勾选「我已手动核对/查重」。 |
| | **导出格式与方式** | 下载 Word/PDF 等。 |
| **用户交互动作** | 进入步骤 6 后，等待或触发系统执行引用核对与查重（第三方接口） | 无需用户自行去外部系统操作。 |
| | 查看系统展示的引用核对结果（如不可溯源明细）与查重报告（重复率等） | 知情后确认。 |
| | 确认已查看后，点击「下载论文」 | 完成主流程。 |
| | 可选：根据结果回到步骤 4 修改不可溯源引用或降重后再导出 | 支持从步骤 6 回看或回到步骤 4。 |

**质量关联**：引用核对与查重由**系统调用第三方接口**完成，用户只需查看结果并确认，无需手动查重或逐条核对；既保证稿子经过核对与查重，又避免用户操作负担。

---

## 七、小结：为保质量不能省的用户动作与信息

| 步骤 | 不能省的用户提交信息 | 不能省的用户交互动作 |
|------|------------------------|------------------------|
| 1 选题 | 学科、研究主题、论文类型；可选：学校/格式模板 | 选择 + 输入/选择 + 可选选学校或格式 + 下一步 |
| 2 大纲 | 对大纲的「采用」或「微调后的标题」 | 查看 + 确认/微调 + 下一步 |
| 3 资料 | 有料：上传文献；无料：明确选择不上传并知悉风险 | 选择有料/无料 + 有料时确认文件列表 + 下一步 |
| 4 正文 | 每章生成请求 + 对内容的审阅/修改 + 处理不可溯源引用 | 按章选择 → 生成本章 → 查看/改并处理系统标出的不可溯源引用 → 全部完成后下一步（若选一键生成须先见警示，生成后仍做引用校验） |
| 5 润色 | 是否执行润色、是否降重（可默认勾选） | 点击润色 + 可选勾选 + 下一步 |
| 6 导出 | 查看系统返回的引用核对结果与查重结果 + 导出意图 | 系统调用第三方引用核对与查重接口 → 用户查看结果并确认 → 下载（无需用户手动查重） |

界面可以简化文案、减少分叉、用默认值减少选择，但**上述「需提交信息」和「必做动作」若被自动跳过或替代，论文质量的可控性会下降**。设计时应在「简单」与「必要输入/确认」之间取得平衡。

## 八、场景：用户已有 10 篇参考文献 —— 傻瓜式成稿与差异化

假设用户**已经准备好 10 篇参考文献**（如从知网/万方导出题录或上传 PDF 列表），要尽量少操作就得到一篇论文，且**同一批 10 篇文献被不同用户使用时，生成的论文不能雷同**。下面给出流程设计和差异化策略。

---

### 8.1 傻瓜式流程（以「10 篇参考文献」为起点）

| 步骤 | 用户要做的（极简） | 系统要做的 |
|------|--------------------|------------|
| **1 导入参考文献** | 上传 1 个文件（Bib/EndNote/Excel 等）或粘贴 10 条题录/DOI；或从「我的文献库」勾选 10 篇 | 解析并固定为本次写作的 10 篇 ref，校验数量与必填字段（标题、作者、年份等） |
| **2 定角度 + 选类型** | **选一个「研究角度」**（见下）；选论文类型（本科/硕士/期刊）；**可选**选学校或格式模板（与主流程一致，导出时按该校/格式） | 根据角度 + 类型生成大纲；角度写入 prompt；格式用于步骤 5、6 |
| **3 确认大纲** | 看大纲，点「就用这个」或微调几处标题后确认 | 锁定大纲，作为按章生成的依据 |
| **4 生成正文** | 推荐按章点「生成本章」并逐章确认；若提供「一键生成全部章节」，须在 UI 上警示「一键模式跳过逐章确认，质量可控性降低」。等；可改字、重生成某一章；**处理系统标出的不可溯源引用** | 每章生成时必带：10 篇 ref（且**正文引用仅限该 10 篇**）+ 研究角度 + 大纲 + 已生成章节摘要；生成后做**引用溯源校验**，标出无法对应到 10 篇 ref 的引用 |
| **5 润色与导出** | 点「一键润色」→ 选是否降重 → 下一步；步骤 6 系统执行引用核对与查重、用户查看结果并确认后下载 | 同前文设计 |

**用户最少动作**：导入 10 篇 ref → 选 1 个角度 + 选 1 个类型 → 确认大纲 → 生成各章（可一键）→ 润色 → 下载。**不要求用户自己写题目或摘要**：题目可由系统根据「10 篇 ref + 研究角度」生成一个建议，用户可采纳或改一句。

---

### 8.2 为什么「同一批 10 篇文献」会生成不同论文？—— 差异化来源

要让**相同 10 篇参考文献在不同用户下产出不同论文**，必须引入「随用户或随任务变化」的变量，且这些变量要**参与生成**（进 prompt / 进大纲）。仅靠「同一 ref 列表 + 同一题目」会高度雷同，不可取。

下面分**用户必选/必填**与**系统自动**两类，保证可操作且可落地。

---

#### （1）用户侧：必选「研究角度」—— 同一批 ref，不同切入点

在「定角度 + 选类型」一步，**强制用户从若干「研究角度」里选一个**（单选），必要时再给一个短句补充。角度会直接决定论文的**问题意识、论证主线和章节侧重**，从而在同一批文献上写出不同文章。

**示例（通用/经管类）**（可按学科换文案）：

| 角度选项（用户任选 1 个） | 对论文的影响（系统用于 prompt） |
|---------------------------|----------------------------------|
| 聚焦「影响因素」 | 大纲与正文偏重：哪些因素、如何衡量、实证结果；文献综述按「因素—结论」组织 |
| 聚焦「机制与路径」 | 偏重：作用机制、中介/调节、路径分析；综述按「机制链」组织 |
| 聚焦「差异与对比」 | 偏重：不同群体/地区/时期的差异、对比与分类；综述按「对比维度」组织 |
| 聚焦「问题与对策」 | 偏重：现状问题、成因、对策建议；综述按「问题—对策」组织 |
| 聚焦「演进与趋势」 | 偏重：时间线、阶段、未来趋势；综述按「演进脉络」组织 |

- **交互**：一屏展示 5～6 个角度，**大卡片或单选按钮**，选 1 个即确定；可选一个**短句补充**（如「我想突出中小企业」），占位符示例：「可写一句补充（选填）」。
- **质量与差异化**：同一 10 篇 ref，用户 A 选「影响因素」、用户 B 选「机制与路径」，生成的大纲和正文就会明显不同；若再配合下面的系统侧策略，雷同率可进一步降低。

---

#### （2）用户侧：题目/研究问题的生成与可改

- **系统**：用「10 篇 ref 的标题/关键词 + 用户选的研究角度 + 论文类型」生成**一个默认题目 + 一个研究问题句**，展示给用户。
- **用户**：可**直接采用**（零输入）或**改一句**再确认。这样既保证「同一 ref + 同一角度」时题目仍可不同（用户改一句即可），又避免用户从零想题。

---

#### （3）系统侧：大纲与叙述结构的随机/多版本

- **大纲结构**：在「研究角度」确定的前提下，系统可预置 2～3 种**章节排布模板**（如「先理论后实证」vs「先现象后机制」），按**用户 ID 或本次任务 ID 的哈希**选一个，或随机选。同一 ref + 同一角度下，不同用户得到不同章节顺序/侧重，正文自然不同。
- **叙述偏好**：在 prompt 里加一条**随机的「叙述偏好」**（如「偏重批判性综述」vs「偏重归纳性综述」），用任务 ID 做种子，保证同一请求多次生成也会有差异（可用于「重新生成本章」时的多样性）。

---

#### （4）系统侧：生成时的随机性与引用侧重

- **温度/采样**：调用大模型时，**不用 temperature=0**；设为 0.6～0.8 并配合 max_tokens，使同一 prompt 多次生成有合理差异。
- **章节顺序与引用侧重**：同一批 10 篇 ref，在不同任务中可**随机或按规则调整「优先引用顺序」**（如有的章多引用 1、3、5，有的章多引用 2、4、6），避免所有用户都「按同一顺序」综述同一批文献。

（说明：将「任务 ID 写入 prompt」在实际工程中难以带来可区分的语义差异，真正起作用的是研究角度、结构模板和温度，此处不采用任务 ID 入 prompt 的做法。）

---

### 8.3 差异化小结（同一 10 篇 ref → 不同论文）

| 层级 | 手段 | 谁控制 | 作用 |
|------|------|--------|------|
| 必选 | **研究角度**（如影响因素 / 机制 / 对比 / 问题对策 / 演进） | 用户 | 决定问题意识与主线，同一 ref 不同角度 → 不同大纲与正文 |
| 可选 | **题目/研究问题**（系统建议，用户可改一句） | 用户 | 进一步区分主题表述与侧重点 |
| 系统 | **章节结构模板**（2～3 种按用户/任务选） | 系统 | 同一角度下不同排布与侧重 |
| 系统 | **温度 + 叙述偏好** | 系统 | 同一 prompt 下表述与结构仍有差异 |
| 系统 | **引用侧重/顺序**（按章或按任务微调） | 系统 | 同一 ref 在不同文中「出场方式」不同 |

这样，**即使用户 A 和用户 B 上传了完全相同的 10 篇参考文献**，只要两人选的「研究角度」不同，或有一人改了题目/研究问题，再叠加系统侧的结构与随机策略，生成的论文就会在**问题意识、结构、表述和引用侧重**上明显区分，避免雷同。

---

### 8.4 界面与文案上的「傻瓜式」要点

- **步骤 1**：标题直接写「上传或选择你的 10 篇参考文献」，下面一个大拖拽区 + 「支持 Bib/Excel/复制题录」，并显示「已选 X/10 篇」；满 10 篇后自动亮起「下一步」或允许用户继续到下一步（不足 10 篇时提示「建议凑满 10 篇再生成」）。
- **步骤 2**：标题写「选一个研究角度，定下论文方向」；下列 5～6 个大卡片（角度名称 + 一句话说明），再一行「论文类型：本科 / 硕士 / 期刊」，**可选**「选择学校或格式模板」（与主流程步骤 1 及文末附录「格式模板说明」一致，格式须在本步或步骤 1 选定）；底部展示系统生成的**题目 + 研究问题**，附带「就用这个」和「改一句」。
- **后续步骤**：与前面「六步」一致（确认大纲 → 按章生成 → 润色 → 核查与导出），只是步骤 1 固定为「10 篇 ref」，步骤 2 固定为「角度 + 类型 + 题目/研究问题」。

这样，用户**已有 10 篇参考文献**时，只需：**导入 ref → 选角度 → 选类型 → 确认题目与大纲 → 生成 → 润色 → 下载**，即可傻瓜式得到一篇论文，且通过「角度 + 题目/问题 + 系统随机与结构」保证不同用户用相同 ref 时论文不会一样。

---

## 附录：格式模板与不同学校/单位要求

不同大学、院系对毕业论文的**版式与格式**要求差异很大，若导出后再手工改格式，既费时又易出错。**格式须在步骤 1 选定**，步骤 5 润色与步骤 6 导出均按此处所选处理，时序上不推迟到步骤 6，否则步骤 5 的格式检查将无依据。

**常见差异项与系统支持方式**：

| 常见差异项 | 说明 | 系统可支持方式 |
|------------|------|----------------|
| **参考文献著录规范** | 国标 GB/T 7714、APA、MLA、各校自定顺序与标点等 | 预设「国标/通用」+ 按学校选择（学校模板内绑定该规范）；或单独选项「参考文献格式」 |
| **标题层级与编号** | 一级/二级/三级标题的字体、字号、编号样式（如 1 / 1.1 / 1.1.1 或 一、（一）） | 每个学校模板内预定义；导出时按模板套用 |
| **页眉页脚** | 页眉是否写学校名、论文题目、页码位置与样式 | 学校模板内预定义 |
| **正文与版心** | 字体、字号、行距、段前段后、页边距 | 学校模板内预定义 |
| **摘要与目录** | 中英文摘要格式、目录是否自动生成、是否单独页 | 学校模板内预定义 |

**产品设计建议**：

- **步骤 1 选定格式**：提供「学校列表」（可搜索）、「国标/通用」、「自定义」。选学校即绑定该校预设模板；选「自定义」时，**须在本步内**上传单位 .dotx 或选择参考文献标准（如 GB/T 7714），**不推迟到步骤 6**。若用户选「自定义」但本步未上传/未选，则步骤 5、6 按「国标/通用」执行（或步骤 5 的格式检查跳过，仅做语气与术语，导出时用国标/通用）。
- **模板维护**：系统维护「学校 ID → 格式模板」映射；模板包含版式（页眉页脚、标题样式、正文样式）与参考文献输出格式（如 GB/T 7714-2015 顺序编码制）。可优先覆盖常见高校，其余用「国标/通用」；支持运营侧持续补充学校模板。
- **润色与导出**：步骤 5「检查格式」与步骤 6「下载论文」**均读取步骤 1 已选定的学校/格式**，按对应模板生成，避免多校混用或时序不一致。
