# 傻瓜式 AI 写论文 — 后台管理网站功能清单

> 基于主流程（六步 + 订单 + 改稿）、格式模板、查重/引用对接等设计，列出后台管理端应具备的功能模块与子项。

---

## 一、订单与任务

| 功能 | 说明 |
|------|------|
| **订单列表** | 按时间、状态（生成中 / 写作完成 / 失败）、用户、标题筛选；支持导出。 |
| **订单详情** | 查看订单信息、论文标题、创建/完成时间、支付信息（若有）、当前状态；关联的「生成任务」进度与日志。 |
| **订单状态与操作** | 人工改状态（如失败→重试）、标记异常、备注；失败订单支持「重新触发生成」或转客服。 |
| **生成任务 / 队列** | 查看后台生成任务队列（排队中、执行中、完成、失败）；单任务日志（按章生成进度、报错）；支持重试、取消。 |

**对应主流程**：步骤 4 支付完成创建订单，步骤 5 后台生成论文；订单条目展示论文生成状态，仅「写作完成」可进入在线改稿。

---

## 二、用户

| 功能 | 说明 |
|------|------|
| **用户列表** | 按注册时间、最近活跃、ID、手机/邮箱筛选；基础信息（脱敏）。 |
| **用户详情** | 基本信息、订单数、完成数、失败数；可选：常用学科/格式、最近订单列表。 |
| **禁用/风控** | 违规用户禁用、限流（如单日生成上限）；操作留痕。 |

---

## 三、格式模板与学校

| 功能 | 说明 |
|------|------|
| **学校/格式模板列表** | 按学校名称、学历、是否启用筛选；支持搜索。 |
| **新增 / 编辑模板** | 绑定学校（或「国标/通用」）、学历；上传或配置 .dotx、页眉页脚、标题层级、参考文献著录规范等；启用/停用。 |
| **未找到学校 / 用户提交** | 用户未找到学校时，通过**客服 联系与反馈**（或扫码加客服好友）联系客服，**向客服上传规范要求文档**（如学校格式规范、.dotx）；客服/运营在后台维护，补充该校模板并启用。列表可展示：用户提交的学校名、上传的规范文档、处理状态（待补充 / 已补充 / 已忽略）；补充后可关联到新模板并通知用户（若实现）。 |
| **自定义模板审核** | 若支持用户上传 .dotx：列表与审核状态（通过/拒绝），通过后该用户可选自己的模板。 |

**对应主流程**：步骤 1 按学校搜索、未找到时兜底「联系客服」；用户向客服上传规范要求文档，由客服/运营在后台维护并补充学校模板；附录格式模板与运营补充学校模板。

---

## 四、内容与合规

| 功能 | 说明 |
|------|------|
| **协议与文案管理** | 服务条款、隐私政策、步骤 1 用户协议勾选文案；各步骤说明文案、错误提示文案（可配置，便于合规与运营话术统一）。 |
| **内容抽检 / 违规** | 按订单或论文抽样查看生成内容；标记违规、下架、通知用户；若接入了内容安全接口，可展示机审结果。 |
| **敏感词 / 策略** | 敏感词库、违规策略配置（如禁止某些主题、引用约束说明）；与生成或导出前的策略一致。 |

---

## 五、文献、查重与引用

| 功能 | 说明 |
|------|------|
| **查重 API 配置** | 第三方查重 API 或自定义查重模块 API 的地址、密钥、限流；开关与降级策略（当前文档为「不考虑降级」时可仅保留主配置）。 |
| **引用核对配置** | 第三方引用核对接口或自建「引用与 ref 列表比对」服务地址、开关。 |
| **文献库（可选）** | 若自建推荐文献库：题录/文献的增删改查、按学科/主题打标、上下架，供步骤 2 系统推荐。 |

**对应主流程**：步骤 2 文献必选与系统推荐；步骤 6 查重、引用核对与导出前确认。

---

## 六、商品与支付（若已商业化）

| 功能 | 说明 |
|------|------|
| **商品 / 版本管理** | 基础版、AI 率保障版、无限改稿版等；价格、描述、赠品（在线改稿、查重、AIGC 检测等）；上下架。 |
| **支付与退款** | 支付渠道配置；订单支付记录、退款申请与处理。 |
| **优惠与资源包** | 改稿次数、卡券、活动配置（若与主流程「购买资源包」等一致）。 |

**对应主流程**：步骤 4 预览与支付、版本/套餐选择。

---

## 七、数据统计与看板

| 功能 | 说明 |
|------|------|
| **订单与转化** | 订单量、完成率、失败率；按日/周/月；步骤转化（如步骤 1→2→3→4→5→6 各步完成数）。 |
| **生成质量与性能** | 平均生成时长、按章耗时；失败原因分布（便于排查模型/接口问题）。 |
| **收入（若有支付）** | 收入汇总、按版本/套餐分布。 |
| **用户与使用** | 新增用户、活跃用户、人均订单；学科/格式选用分布（便于优化模板覆盖）。 |

---

## 八、系统与权限

| 功能 | 说明 |
|------|------|
| **管理员与角色** | 账号、角色（如超管、运营、客服、只读）；权限与菜单绑定。 |
| **操作日志** | 关键操作（订单状态变更、模板上下架、配置修改、用户禁用）的谁、何时、做了什么。 |
| **系统参数** | 全局开关（如维护模式、单日生成上限、单用户限流）；基础配置（如默认字数建议、文献数量建议）与主流程一致。 |

---

## 九、功能模块汇总表（按优先级建议）

| 优先级 | 模块 | 核心子项 | 说明 |
|--------|------|----------|------|
| P0 | 订单与任务 | 订单列表/详情、状态、生成任务与重试 | 运营与排查必备 |
| P0 | 格式模板与学校 | 模板列表/编辑、未找到学校提交与处理 | 与主流程步骤 1、附录强相关 |
| P0 | 系统与权限 | 管理员、角色、操作日志 | 安全与审计 |
| P1 | 用户 | 用户列表/详情、禁用与限流 | 风控与运营 |
| P1 | 文献/查重/引用 | API 配置、引用核对配置 | 与步骤 2、6 一致 |
| P1 | 协议与文案 | 协议与步骤文案配置 | 合规与话术统一 |
| P2 | 数据统计 | 订单转化、完成率、失败原因、收入 | 产品与运营决策 |
| P2 | 商品与支付 | 版本/套餐、支付与退款 | 若已商业化 |
| P3 | 内容与合规 | 抽检、敏感词、违规策略 | 内容安全与合规增强 |

---

以上清单可直接作为后台管理网站的一级/二级菜单与需求依据；实现时可按 P0 → P1 → P2 分阶段上线。
